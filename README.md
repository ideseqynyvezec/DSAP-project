# 虛擬貨幣與外匯「三角套利」機會尋找器

## Proposal Report

### 動機與目標
我以前就常常聽說過外匯與虛擬貨幣市場中的「三角套利」策略，覺得這種利用市場定價無效率來獲利的機制非常神奇，一直想要親自探索它的真實性與運作邏輯。藉由這次專題的機會，我希望能將課堂上學到的「資料結構」與「進階程式設計」理論轉化為實際的工具。本專案的目標是開發一款輕量級的套利路徑尋找器。系統將讀取多種貨幣間的匯率，透過演算法快速偵測是否存在套利空間，並輸出具體的交易路徑與預期報酬率。

### 預期功能
本專案將專注於最核心的邏輯實作：
1. 讀取基礎匯率：讀取一份包含 4~5 種貨幣匯率的本地靜態 CSV 或 JSON 檔案。
2. 建構基礎圖形：將讀取的資料轉換為基本的有向圖（Directed Graph）結構。
3. 基礎三角套利偵測：實作基礎的搜尋邏輯（深度優先搜尋 DFS），嚴格限制只尋找「3個幣種」的套利循環（例如：TWD -> USD -> JPY -> TWD）。
4. 文字結果輸出：在終端機單純印出找到的路線以及相乘後的匯率數值。

### 使用技術
* 程式語言：Python / C++
* 資料結構實作：使用基本的 Hash Map 或 Adjacency List 來實作 Graph。
* 開發工具：Git / GitHub 進行版本控制。

### 時程規劃
* Week 7-8：繳交 Proposal，建立 GitHub Repo，並準備一份只有 5 種貨幣的假資料。
* Week 9-10：將假資料讀入程式，實作 Graph 資料結構，並寫出尋找長度為 3 的 DFS 搜尋邏輯。
* Week 11：程式能成功讀取資料並跑出結果（即使介面很簡陋也沒關係），發布 Prototype Release。
* Week 12-13：不增加新功能。專注於修復程式 Bug，處理極端狀況（例如找不到套利路徑時程式不會崩潰），並補齊程式碼註解。
* Week 14：撰寫 README.md 的最終報告，並開始錄製 5 分鐘 Demo 影片。
* Week 15：確認所有文件與影片連結無誤，發布 Final Report Release。

### 與課程的關聯
1. 圖論的應用：專案的核心基礎是一個有向圖 (Directed Graph)。市場上的「貨幣」對應圖中的「節點 (Vertices)」，而「匯率」則對應「有向邊 (Edges)」，完美將現實數據映射到資料結構上。
2. 深度優先搜尋 (DFS)：利用帶有深度限制（Depth=3）的 DFS 演算法，在圖形中尋找能回到起點且權重相乘大於 1 的特定路徑。
3. 物件導向程式設計 (OOP)：在進階程式設計的層面，我會運用封裝（Encapsulation）的概念，將圖的建立、節點的連接以及搜尋演算法等功能模組化，確保程式碼的可讀性與可維護性。
